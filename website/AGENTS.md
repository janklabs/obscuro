# website/

Next.js 16.2.4 + React 19.2.4 + fumadocs 16.8.5 docs site. Deployed to Cloudflare via OpenNext. **Versioned independently from the Go CLI** — `.github/workflows/release.yml` excludes `website/**` from CLI version bumps.

## Stack pins (do not casually upgrade)

| Pkg                      | Version | Why pinned                                                                          |
| ------------------------ | ------- | ----------------------------------------------------------------------------------- |
| `next`                   | 16.2.4  | App Router; APIs differ from older training data (see banner above)                 |
| `react` / `react-dom`    | 19.2.4  | RC features used in `_components/`                                                  |
| `fumadocs-{core,mdx,ui}` | 16.8.5  | Docs framework — version triplet must match                                         |
| `@opennextjs/cloudflare` | 1.19.5  | Cloudflare Workers adapter — `next start` won't be used in prod                     |
| `tailwindcss`            | 4.x     | New `@tailwindcss/postcss` pipeline; **no `tailwind.config.js`** — config is in CSS |
| `pnpm`                   | 10.33.2 | `preinstall` hook rejects npm/yarn                                                  |

## Commands

```bash

pnpm lint                 # eslint --max-warnings=0  ← zero warnings enforced in CI
pnpm format:check         # prettier --check
pnpm format:write         # prettier --write
pnpm preview              # opennextjs build + local preview
pnpm deploy               # opennextjs build + cloudflare deploy (CI only; needs Infisical OIDC)
```

## CI (`.github/workflows/website.yml`)

Triggers on `website/**` changes only. Runs `pnpm install --frozen-lockfile` → `pnpm lint` → `pnpm format:check`. Deploys on `main` after secrets pulled from Infisical.

## Conventions specific to this site

- **Theme**: `vitesse-dark` for BOTH light and dark mode (intentional, see `source.config.ts`). Do not introduce a separate light theme without product sign-off.
- **Code blocks**: rehype-code processes them at build time — no client-side highlighter.
- **Imports sorted by `@ianvs/prettier-plugin-sort-imports`** — let prettier own ordering; manual reordering will be reverted.
- **Tailwind v4**: utility classes only; styles live in `globals.css` via `@import "tailwindcss"`. No `tailwind.config.js` exists.
- **`shadcn/ui`** components are auto-scaffolded into `src/components/ui/` — re-run `pnpm dlx shadcn@latest add <name>` instead of editing.

## Anti-patterns

- **Do not** write to `.source/` — regenerated on every `next dev`/`build`.
- **Do not** import from `next/document`, `pages/_app.tsx`, or other Pages Router APIs — App Router only.
- **Do not** add `"use client"` to MDX content; use it only in `src/app/_components/*.tsx` interactive bits.
- **Do not** bump fumadocs packages individually; the three (`core`, `mdx`, `ui`) must move together.
- **Do not** use `npm`/`yarn` — `preinstall` hook fails the install (`only-allow pnpm`).
- **Do not** ship secrets via `next.config.ts` env — production runs on Cloudflare Workers; use `wrangler.jsonc` bindings.
- **Do not** treat docs site changes as CLI changes — they will not bump the Go binary version.

## Notes

- Custom fonts live in `src/app/fonts/` (Satoshi, ClashDisplay, ClashGrotesk, IosevkaSS02). Loaded via `next/font/local` in `src/app/layout.tsx`.
- Search uses `flexsearch` via `src/app/api/search/route.ts` (fumadocs default endpoint).
- `src/lib/source.ts` is the single source of truth for the docs tree — page lookups, sidebar, search all derive from it.
